
# Inference Layer: Ollama Setup

This folder contains the inference layer for the task management agent, using **Ollama** as the primary LLM runtime.

---

## **Folder Structure**
```
inference/
├── ollama/               # Ollama runtime configuration
│   ├── pull_model.sh     # Pulls the Mistral-7B model
│   └── start_server.sh   # Starts the Ollama server
├── client/               # Test scripts
│   ├── test.py           # Python script to test the Ollama endpoint
│   └── test_ollama.sh    # Shell script to test the Ollama endpoint
├── logs/                 # Logs generated by Ollama
└── README.md             # This file
```

---

## **Setup Instructions**

### **1. Install Ollama**
Run the following command to install Ollama on your system:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

---

### **2. Pull the Mistral-7B Model**
Navigate to the `ollama/` directory and run:

```bash
cd ollama
./pull_model.sh
```
This will download the `mistral:7b-instruct` model.

---

### **3. Start the Ollama Server**
Run the following script to start the Ollama server:

```bash
./start_server.sh
```
- The server will start on `http://localhost:11434`.
- Logs will be saved to `../logs/ollama.log`.

---

## **Testing the Inference Endpoint**

### **Option 1: Using Python Script**
Run the Python test script to validate the Ollama endpoint:

```bash
cd ../client
python test.py
```

### **Option 2: Using Shell Script**
Alternatively, use the shell script to test the Ollama endpoint:

```bash
cd ../client
./test_ollama.sh
```

### **Expected Output**
Both scripts will send a test prompt to the Ollama server. The response should look like:

```json
{
  "model": "mistral:7b-instruct",
  "response": "Test response from Ollama..."
}
```

---

## **Troubleshooting**

### **1. Ollama Server Not Starting**
- Ensure Ollama is installed correctly:
  ```bash
  ollama --version
  ```
- Check for port conflicts:
  ```bash
  lsof -i :11434
  ```

### **2. Model Not Found**
- Verify the model was pulled successfully:
  ```bash
  ollama list
  ```

### **3. Permission Issues**
- Ensure scripts are executable:
  ```bash
  chmod +x ollama/*.sh client/test_ollama.sh
  ```

---

## **Notes**
- **Ollama is the primary runtime**: This setup does not include Lemonade or AMD-specific optimizations.
- **Logs**: The `logs/ollama.log` file will be created automatically when you run `start_server.sh` for the first time.
- **Testing**: Use either the Python script (`test.py`) or the shell script (`test_ollama.sh`) to test the Ollama endpoint.
