# Flowstate

Flowstate is an always-on workflow orchestration library for teams whose work is spread across chat, docs, email, and other systems rather than a single project tracker.

It solves three core problems:

1. Work fragmentation: important tasks and dependencies stay buried in conversations and files.
2. Stale planning systems: manually maintained trackers drift from real execution state.
3. Slow blocker resolution: teams discover bottlenecks too late and react manually.

How Flowstate solves this:

- Ingests activity from multiple sources and normalizes it into machine-usable events.
- Extracts tasks and dependencies, enriches ownership/deadlines, and builds a live dependency graph.
- Computes intelligence signals (critical path, bottlenecks, stale blockers) and triggers downstream actions through connectors and agent tools.

Flowstate is built by Siya and Srishti.
